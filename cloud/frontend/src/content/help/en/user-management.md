Under **Users**, hire company and organisation admins manage cloud access for staff and customers.

## Create users

Required fields when creating:

- **Name** and **email** (login)
- **Role** — determines visibility and permissions (see **Roles and access**)
- **Password** — initially set by the administrator

## Assign organisations

Via the organisation picker you define which mandates the user can access:

- **Hire company admin** — all organisations of the hire company
- **Organisation admin** — only assigned organisations (edit)
- **Member** — only assigned organisations (operational)

Without organisation assignment a user sees no org-specific data.

## Event admin PIN (optional)

For users with organisation assignment an **event admin PIN** can be set:

- Numeric code (max. 6 digits)
- Enables admin functions on the Pi within assigned organisations
- When editing the PIN can be changed or removed

## Edit users

In edit mode:

- Adjust name, email, and role
- Password is not shown — set a new one only if needed
- Update organisation assignments
- Change or remove event admin PIN

## Tips

- Assign roles sparingly — see permission matrix in **Roles and access**
- Each user needs a valid email for login
- Organisation admins only see users in their organisation(s)
