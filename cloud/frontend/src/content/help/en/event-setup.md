An event typically moves through the statuses **Configuration → Live → Completed**. Most settings are made in edit mode.

## Master data

- Set name, date range, and currency
- Enable payment types (cash, TWINT, card, etc.)
- Status controls whether the event is visible on Pi devices

In **Configuration** status you can optionally use a hosted cloud Pi sandbox for testing.

## Stations

Stations group points of sale (e.g. bar, food). Each station can have its own printers and stock levels.

## Kitchen monitor

Optional: kitchen screens with order overview per station.

## Event waiters

Assign waiters to the event. Only assigned waiters can sign in on the Pi.

## Cash registers

When registers are enabled: configure register floats and shift settlements.

## Vouchers

Optional: define voucher types and limits for the event.

## App layouts

Choose which articles appear on which waiter layouts. Per cell you can set label, **color**, articles, and optional vouchers.

If the organisation maintains a **color palette (app layout)**, those colors appear as labeled quick picks when editing a cell. Otherwise pick a color freely with the color picker or hex field.

How to manage the palette: help article **Set up organisation** → Color palette section.

## Receipts

Configure receipt and payment receipt profiles for printers and tickets.

## Stock items

Manage stock per station; sales reduce stock automatically.

## Operations (live)

For running events, sales, collective bills, and transactions are also available.
