Kellner sind org-weite Stammdaten. Jeder Kellner hat einen Namen und eine PIN für die Anmeldung am Pi.

## Kellner anlegen

Unter **Kellner** legen Sie pro Person mindestens an:

- **Name** — wird in Berichten und am Pi angezeigt
- **PIN** — numerischer Code für die Kellner-Anmeldung

Die PIN ist organisationsweit eindeutig. Vergeben Sie keine doppelten PINs.

## Anmeldung am Pi

Kellner melden sich auf dem Pi mit ihrer PIN an. Ohne gültige PIN ist kein Verkauf möglich.

**Wichtig:** Ein angelegter Kellner kann sich erst anmelden, wenn er dem Event zugeordnet ist (siehe **Event einrichten** → Event-Kellner).

## Unterschied zum Event-Kellner

| Bereich | Kellner (Stamm) | Event-Kellner |
|---------|-----------------|---------------|
| Ort | Org-weite Liste | Event-Konfiguration |
| Inhalt | Name, PIN | Zuweisung zum Event |
| Zweck | Stammdaten pflegen | Wer darf an diesem Event verkaufen? |

Legen Sie Kellner einmal im Stamm an und weisen Sie sie pro Event zu.

## Tipps

- Eindeutige PINs vergeben (z. B. keine doppelten Vierer-Kombinationen)
- Kellner vor Eventstart anlegen und im Event zuweisen
- Kellner ohne Event-Zuweisung können sich nicht anmelden — das ist kein Fehler, sondern Absicht
