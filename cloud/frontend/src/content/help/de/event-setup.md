Eine Veranstaltung durchläuft typischerweise die Status **Konfiguration → Live → Abgeschlossen**. Die meisten Einstellungen nehmen Sie im Bearbeitungsmodus vor.

## Stammdaten

- Name, Zeitraum und Währung festlegen
- Zahlungsarten aktivieren (Bar, TWINT, Karte usw.)
- Status über die Statusleiste weiterschalten (Bestätigen speichert den Status sofort; übrige Stammdaten weiterhin über Speichern)

Im Status **Konfiguration** können Sie optional eine gehostete Cloud-Pi-Sandbox für Tests nutzen.

## Stationen

Stationen gruppieren Verkaufspunkte (z. B. Bar, Food). Jede Station kann eigene Drucker und Bestände haben.

## Kitchen Monitor

Optional: Bildschirme für die Küche mit Bestellübersicht pro Station.

## Event-Kellner

Weisen Sie Kellner dem Event zu. Nur zugeordnete Kellner können sich am Pi anmelden.

## Kassen

Bei aktivierten Kassen: Kassenstände und Schichtabschlüsse konfigurieren.

## Gutscheine

Optional: Gutscheinarten und Limits für das Event definieren.

## App-Layouts

Legen Sie fest, welche Artikel auf welchen Kellner-Layouts erscheinen. Pro Zelle können Sie Bezeichnung, **Farbe**, Artikel und optional Gutscheine setzen.

Wenn die Organisation eine **Farbpalette (App Layout)** pflegt, stehen diese Farben beim Bearbeiten einer Zelle als beschriftete Schnellauswahl zur Verfügung. Andernfalls wählen Sie die Farbe frei per Farbwähler oder Hex-Eingabe.

Details zur Pflege der Palette: Hilfeartikel **Organisation einrichten** → Abschnitt Farbpalette.

## Belege

Bon- und Zahlungsbeleg-Profile für Drucker und Quittungen einstellen.

## Lagerartikel

Bestände pro Station verwalten; Verkäufe reduzieren den Lagerstand automatisch.

## Betrieb (Live)

Bei laufenden Events stehen zusätzlich Umsatz, Sammelrechnungen und Transaktionen zur Verfügung.
