Waiters are organisation-wide master data. Each waiter has a name and a PIN for signing in on the Pi.

## Create waiters

Under **Waiters** create at least per person:

- **Name** — shown in reports and on the Pi
- **PIN** — numeric code for waiter login

The PIN must be unique within the organisation. Do not assign duplicate PINs.

## Sign in on the Pi

Waiters sign in on the Pi with their PIN. Without a valid PIN no sales are possible.

**Important:** A created waiter can only sign in after being assigned to the event (see **Set up an event** → Event waiters).

## Difference from event waiters

| Area | Waiters (master) | Event waiters |
|------|------------------|---------------|
| Location | Org-wide list | Event configuration |
| Content | Name, PIN | Assignment to event |
| Purpose | Maintain master data | Who may sell at this event? |

Create waiters once in the master and assign them per event.

## Tips

- Assign unique PINs (e.g. avoid duplicate four-digit combinations)
- Create waiters before the event and assign them in the event
- Waiters without event assignment cannot sign in — this is intentional, not an error
